# FitFlow PRD (Product Requirements Document)

> 작성: 김민수 (PM) | 최종 수정: 2025-12-15  
> 상태: Draft v0.9

## 1. 배경

FitFlow는 수도권 5개 센터를 운영하는 피트니스 체인이다. 현재 각 센터별로 엑셀과 수기 장부로 회원 관리를 하고 있으며, 이를 통합 관리 시스템으로 전환한다.

### 1.1 목표
- 회원 정보 통합 관리
- 회원권(멤버십) 상태 자동 관리
- PT 세션 스케줄링 및 트레이너 매칭
- 기본 통계 대시보드 제공
- 간단한 챗봇 FAQ 기능

### 1.2 범위
- **In-scope:** 회원 CRUD, 회원권 관리, PT 세션 관리, 트레이너 매칭, 통계, 챗봇
- **Out-of-scope:** 결제 연동, 모바일 앱, 락커 관리

## 2. 사용자

| 역할 | 설명 |
|------|------|
| 센터 매니저 | 회원 등록, 회원권 발급, 통계 조회 |
| 트레이너 | PT 세션 관리, 스케줄 확인 |
| 본사 관리자 | 전체 센터 통계 조회 |

## 3. 기능 요구사항

### 3.1 회원 관리
- 회원 등록 시 필수: 이름, 전화번호, 센터
- 회원 상태: active, inactive, frozen
- 센터 이관 가능 (이관 이력 보존)

### 3.2 회원권(멤버십) 관리
- 종류: 1개월, 3개월, 6개월, 12개월
- 회원권 동결(freeze) 가능: 최대 30일/년
- 만료일 자동 계산
- **환불 시 잔여 일수 기준 일할 계산**
  - 환불 금액 = 잔여 횟수 x 단가

### 3.3 PT 세션
- 패키지 단위 구매 (10회, 20회, 30회)
- 세션 상태: scheduled, completed, cancelled, no_show
- 무료 체험 세션 제공 (최대 3회)

### 3.4 트레이너 매칭
- 회원의 운동 목표에 따라 트레이너 추천
- 전문 분야: 체중감량, 근력강화, 재활, 체형교정, 스포츠

### 3.5 통계
- 월별 신규 회원 수
- 센터별 활성 회원 수
- PT 세션 소화율
- 매출 요약

### 3.6 챗봇
- 간단한 FAQ 응답
- 회원 잔여 일수/횟수 조회
- 영업시간, 위치 안내

## 4. 비기능 요구사항

### 4.1 환불 정책
- 이용 기간 7일 이내: 전액 환불
- 7일 초과: 잔여 일수 비례 환불, **위약금 10% 적용**
  - 환불 금액 = (잔여 일수 / 전체 일수) x 총 금액 x 0.9

### 4.2 데이터
- 시간대: 서버는 UTC, 비즈니스 기준 KST(+9)
- 소프트 삭제 적용

### 4.3 성능
- API 응답 500ms 이내
