# 인수인계 노트

> 작성: 이준혁 (이전 담당 개발자)  
> 날짜: 2026-01-20

## 현재 상태

MVP 수준까지 개발 완료. 기본 CRUD와 주요 비즈니스 로직이 구현되어 있으나, 엣지 케이스 처리가 미흡한 부분이 있습니다.

## 알려진 이슈

### 높은 우선순위
1. **환불 계산 로직** — PRD 초안과 최종 비즈니스 규칙 사이에 기준이 다릅니다. 현재 코드는 PRD 4.1절 기준인데, PM 김민수님이 마지막에 뭐라고 했는지 기억이 가물가물합니다. `business_rules.md`가 최종 확정본이니 그걸 기준으로 맞춰주세요.

2. **통계 API** — 월별 신규 회원 집계가 뭔가 이상한 것 같은데 확인이 필요합니다. 테스트 데이터에서 1월 말 가입자가 가끔 2월로 넘어가는 현상이 있었습니다.

### 중간 우선순위
3. **PT 잔여 횟수** — 무료 체험 관련해서 뭔가 빠진 것 같은 느낌인데, 시간이 없어서 확인을 못했습니다. 체험 세션 처리 로직을 한번 봐주세요.

4. **트레이너 매칭** — 현재 너무 단순하게 구현되어 있습니다. 특정 트레이너에게 몰리는 문제가 있을 수 있어요.

### 낮은 우선순위
5. **챗봇** — 키워드 기반이라 한계가 있습니다. 유의어 처리가 안 되어 있어서 개선이 필요합니다.

6. **데이터 모델 문서** — 마지막에 추가한 테이블이 문서에 반영이 안 됐을 수 있습니다. 코드 기준이 맞습니다.

## 시드 데이터
- 다양한 엣지 케이스를 테스트하기 위해 시드 데이터를 좀 다양하게 넣어놨습니다
- 방문 이력이 전혀 없는 회원, 동결 이력이 있는 회원 등이 포함되어 있습니다
- 이관 회원도 10명 정도 있는데, 센터별 통계에서 올바르게 집계되는지 확인이 필요합니다
- **데이터 품질 주의:** 초기에 기존 엑셀 데이터를 마이그레이션할 때 급하게 넣은 부분이 있어서, 테이블 간 데이터가 안 맞을 수 있습니다. 특히 회원 상태 필드가 실제 회원권 상태와 다른 경우가 있을 거예요. 정리가 필요한데 시간이 없었습니다.

## 코드 구조 메모
- `services/` 디렉토리에 비즈니스 로직을 분리하려고 했는데, 일부는 라우터에 직접 들어간 부분도 있습니다
- 테스트는 시간 부족으로 기본 CRUD만 작성했습니다. 비즈니스 로직 테스트가 필요합니다
- `membership_calculator.py`에 회원권 관련 계산 로직이 모여 있습니다
